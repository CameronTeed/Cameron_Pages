About the Project

Built with C++ on the G++ compiler for linux. This aplication outputs captured network packets for a set duration of time to a txt file. Testing was done with python, by broadcasting packets to the network and capturing them with the application. The python script then compared the output of the application to the packets sent, and verified that the application was working correctly.

Key Features

  • File I/O
  • Testing and Quality Assurance
  • Python Scripting
  • Version Control
  • OOP
  • Sockets

How it works

Uses the linux socket library to capture packets, and outputs them to a file. The application is run from the command line, and takes in a duration of time to capture packets for, and an output file name.

Project Structure

The project is organized into modular components, following best practices for C++ development. Key functionalities are encapsulated using object-oriented programming (OOP) principles, promoting code reusability and maintainability. Version control is implemented with Git, ensuring a systematic approach to project management and collaboration.

Analyzing Packets

In order to capture packets, and output them to a file, we must understand how packets are structured. For IPv4 packets are made up of a header, a payload, and sometime a trailer. The header contains information about the packet, such as the source and destination IP address, and the payload contains the data being sent.

Testing

Unit tests were conducted using google test (gtest), integration and functional tests were conducted using python, where data packets were broadcasted and searched for in the output file created by the network analyzer. Valgrind was used to ensure there were no memory leaks.
